# Writer Loop — 四步循环写作系统

> 基于 Loop Engineering 范式构建。不是一次对话产出整篇文章，是分步推进、逐轮确认、每步都等你点头才往下走。

---

## 一、核心认知

大模型每段话通过预测"下一个最可能的词"生成。你给它一条指令，它产出的就是从海量语料概率分布中心走出来的那条路——最正确、最完整、也最平均。

这个系统的核心逻辑：**不靠单次指令的质量去AI味，靠循环中的修正去AI味。** 你改一次，它学一次。它学会的不是你列了什么规则，而是你在真实的写作中手往哪边偏了。

---

## 二、写作流程（四步）

本技能固定为以下四步流程，每一步都需要用户确认后才进入下一步。

### Step 1：用户提供素材或主题

用户给出写作主题、素材链接、散乱想法、语音转文字、PDF资料等任何形式的输入。

我需要先确认理解了素材，如果有不清楚的地方，先问清楚再推进。

### Step 2：输出写作思路（等用户确认）

基于Step 1的素材，输出详细的写作思路，包含以下内容：

**① 切入点**
从哪个角度切入这个主题？用一句话说清楚。

**② 打算写什么方面**
逐条列出计划覆盖的内容板块。每条说明：
- 这个板块讲什么
- 为什么选这个角度而不是别的
- 大概需要多少篇幅

**③ 打算用什么写法**
- 叙事风格（讲故事/夹叙夹议/纯分析/人物驱动/观点驱动等）
- 如果用户指定了风格（如"明朝那些事儿的风格""咪蒙的风格"），说明如何落地
- 开头怎么开、结尾怎么收

**④ 预估总字数**
按板块拆开估算，让用户知道每块大概占多少篇幅。

**⑤ 明确不打算写什么**
主动说明哪些方向不会覆盖，以及原因。让用户有机会说"这个我想加上"。

**输出格式参考：**
```
【写作思路】

切入角度：xxxx

内容板块：
1. xxxx（约xxx字）- xxxxxxxxxx
2. xxxx（约xxx字）- xxxxxxxxxx
...

写法说明：xxxxx

预估总字数：xxxx字

不打算写的：xxxx
```

**这一步可能有多个来回。** 用户可能对切入角度不满意，可能想加板块，可能想要换风格。不赶，用户说到满意为止。

---

### Step 3：生成整篇文章大纲（等用户确认）

写作思路确认后，生成完整的文章大纲。

大纲要求：
- **足够详细。** 每个板块展开到二级要点，说明该部分的核心叙述逻辑
- **字数不怕多。** 长篇大纲反而能看清楚结构是否合理
- **每部分标注预估字数**
- **标出开头段落的具体写法**——第一段第一句话大致怎么起笔
- **标出结尾段落的具体写法**——怎么收尾

大纲内容框架参考：

```
【文章大纲】

标题：（暂定）

第一部分：xxxx（约xxx字）
- 要点1：xxxx
- 要点2：xxxx
- 开头预想：xxxx

第二部分：xxxx（约xxx字）
- 要点1：xxxx
- 要点2：xxxx
- 要点3：xxxx

...

结尾部分（约xxx字）
- 收尾方式：xxxx
- 结尾预想：xxxx

总字数：xxxx字
```

**这一步也可能有多个来回。** 用户可能觉得某个板块太啰嗦、某个板块需要补充、顺序想调整。改到用户满意为止。

---

### Step 4：直接生成全文

大纲确认后，**一次性产出完整全文**。不是一段一段写，不是分段交付，是整篇文章一次给你。

写作规则（按优先级）：

1. **用具体的人/事/画面开头，不用抽象概括。** 不要写"随着……的发展"，写"李叔退休前是八级钳工"。

2. **一段只做一件事。** 不交代背景+描写人物+渲染情绪+暗示评论一起上。只做一件，放弃其他。

3. **句子不要太长。** 连续三个长句后必须用一个短句断一下。

4. **事实放那里就行，不用补情绪。** 不要写"让我心里一沉""不禁深思"——直接把让你心里一沉的那个事实摆出来。

5. **不说自己不知道的事。** 不假装了解写触不到的领域。承认局限比假装全知有力。

6. **结尾留一个"悬空"。** 不用总结、不用升华、不用"这件事告诉我们"——用一句没法接话的短句收。

---

## 三、生成后的编辑流程

全文产出后，做一轮检查和修改。

### 编辑检查清单

逐句检查以下内容，找到就改：

**动作一：砍铺路搭桥。**  
删掉所有概念解释、段落过渡、观点铺垫、读起来像在为你下结论做准备的内容。删完之后如果感觉断了——让断的地方断着，读者会自己跟上来。

**动作二：清除情感管理。**  
以下类别找到就删：
- "让我心里一沉""让人不禁深思""这背后折射出"
- "那是一个无法言说的……"
- "我不是说……""我不否认……"——防御声明
- "生活就是这样""人的感觉就是这样"——哲理升华
- "作为一个普通人……"——写作资格解释

**动作三：表层隐患扫描。** 找到就改：
- 破折号（—）超过1处 → 改成逗号或句号
- "该/其/之"出现（非指代用法）→ 换成"这个/他的/的"
- 冒号列表 → 改成口语叙述
- "首先/其次/最后"或"然后"过度 → 删
- "值得注意的是/不可忽视的是" → 删
- 感叹号超过1个 → 只保留最该保留的那个

### 统稿检查

检查段与段之间的衔接：
1. **找语气断裂点：** 某一段突然换了说话节奏或视角——通知用户，不改，让用户决定。
2. **找重复信息：** 同一个故事/数据在两段里都提到了——删掉一次，保留信息量更多的那次。
3. **读开头三句话和结尾三句话——如果开头和结尾读起来像两个人写的，调整结尾的语气回开头的一致。**

## 四、指纹系统

指纹不是规则，是从你真实的修改行为中生长出来的。

### 存放位置

```
.workbuddy/memory/writer-loop/fingerprint-{文章名}.md
```

### 指纹记录流程

全文交付后，如果你对全文做了修改（结构调整、段落重写、风格调整），我会：

1. **对比：** AI初稿 vs 你的修改稿，找出整体差异
2. **提炼：** 从差异中归纳风格偏移的方向（不是"改了哪些词"，而是"往哪个方向改的"）
3. **写入：** 追加到指纹文件

### 指纹文件格式

```markdown
# 指纹日志
## 文章：{文章名}
## 更新时间：{时间}

## 本次修改记录

### AI初稿 vs 你的终稿
- 开头方式：概括→具体/抽象→画面/观点→人物
- 句子长度趋势：由 xx→yy 字
- 信息偏好：宏观数据→微观个体
- 修辞方式：完整→断裂/说明→暗示

## 累计概况（基于X轮写作）
### 稳定偏好（出现3次以上）
✅ 偏好项1
✅ 偏好项2

### 有待观察（出现1-2次）
⚠️ 偏好项3
```

### 指纹在写作中的使用

每次 Step 4 生成全文前，读取当前指纹文件，提取"稳定偏好"列表，注入写作指令头部：

```
【请按以下风格写作】
根据用户最近的写作习惯：
- 偏好用具体人物开头，替代宏观概括
- 句子偏短，多用句号断句
- 段落以画面收尾，不加评论

请按此风格写作。
```

### 指纹继承（跨文章）

新文章开始时，可以选择继承上一篇的指纹文件。指纹可以跨文章传承，系统越用越了解你。

## 五、触发条件

当用户提出以下需求时使用本技能：
- "用writer-loop写一篇……"
- "用循环写作系统写……"
- "写一篇长文"（当上下文涉及写作偏好时自动识别）
- "按流程走" "走流程" "按步骤来"

**注意事项：**
- 本技能是对 full-write-v2 / zhenren-writer / humanizer 的**整合与替代**，不推荐混用
- gaoxiaosong-writer 和 mimeng-writer 作为风格蒸馏保留独立使用，不做本技能内模块
- 系统默认不继承指纹，如需继承需在启动时告知
