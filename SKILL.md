# Writer Loop — 循环写作系统

> 基于 Loop Engineering 范式构建。不是"一次对话产出整篇文章"，而是"逐段写、逐段改、逐段沉淀"。AI 负责填料，你负责给灵魂。

---

## 一、核心认知：为什么单次对话做不到人味

大模型是自回归模型，每段话通过预测"下一个最可能的词"生成。你给它一条指令，它产出的就是从海量语料概率分布中心走出来的那条路——最正确、最完整、也最平均。

技术上说：**单次产出的统计分布天然平滑，而"人味"恰好是统计分布上的毛刺。**

过去所有去AI味的努力——禁用词、角色定义、七大支柱——都是在单次对话框架内对抗这个平滑分布。可以改善，但天花板明显，因为模型只看了一次指令，没有机会"看到你怎么改"。

这个系统的核心逻辑：**不靠单次指令的质量去AI味，靠循环中的修正去AI味。** 你改一段，它学一段。它学会的不是你列了什么规则，而是你在真实的写作中手往哪边偏了。

## 二、两种写作模式

本技能支持两种模式，使用前根据偏好选择。

### 模式A：逐段循环模式（Loop Mode）

适合有时间逐段打磨、追求最高指纹精度的场景。

```
Phase 0: [选题Agent] 产出3个方向 → 你选1个
    ↓
Phase 1: [热身Agent] 四个问题定基调 → 写入状态文件
    ↓
Phase 2: [写作Agent] + 指纹文件 → 产出一段 → 你改 → [指纹Agent] 对比写入 → 下一段
    ↓ (循环直到写成初稿)
Phase 3: [编辑Agent] 初稿通读 → 砍 → 清除情感管理
    ↓
Phase 4: [统稿Agent] 段间衔接检查 → 节奏对位 → 终稿交付
    ↓
Phase 5: 备选标题 x 5
```

介入点：选方向（P0）+ 逐段改段落（P2）+ 确认编辑结果（P3）

### 模式B：完整模式（Full Draft Mode）

大纲确认后，由写作Agent一次性产出完整初稿。你将通读全文并整体修改，指纹从整篇修改中提炼。适合习惯"先写完再整体改"的用户。

```
Phase 0: [选题Agent] 产出3个方向 → 你选1个
    ↓
Phase 1: [热身Agent] 四个问题定基调 → 确认大纲
    ↓
Phase 2: [写作Agent] + 已有指纹 → 一次性产出完整初稿
    ↓
Phase 3: 你通读全文 → 整体修改 → [指纹Agent] 从整篇修改中批量提炼指纹
    ↓
Phase 4: [编辑Agent] 砍三分之一 + 清除情感管理 + 统稿
    ↓
Phase 5: 备选标题 x 5
```

介入点：选方向（P0）+ 修改完整初稿（P3）+ 确认编辑结果（P4）

**模式B下的指纹工作方式：** 不对逐段修改做精细记录，而是在你改完全稿后，对比AI初稿和你的终稿，从整体差异中提取稳定的风格偏移。单次产出的精度低于Loop模式，但长期累积——写三五篇之后——同样能形成有参考价值的个人指纹库。

## 三、Agent 模块（4个角色定义）

### 模块A：选题Agent

**加载时机：** Phase 0

**指令：** 基于给定话题，生成3个完全不同的切入角度。每个角度需包含：一个人物故事线索、一个可感知的画面或场景。不要写选题分析，直接给角度。至少有一个角度是从"一个具体的人"出发，而非从"一个观点"出发。

**输出格式：**
```
角度一：（一句话概括）
人物线索：谁 + 什么事
核心画面：一句能当开头的话

角度二：（同格式）
角度三：（同格式）
```

### 模块B：写作Agent核心指令

**加载时机：** Phase 2（每次写一段之前）

**前置加载：** 当前指纹文件（fingerprint-{文章名}.md）

**核心设定：** 一个在饭局上被人问到这件事的普通人。不是专家，不是记者。放下筷子，想了想，开口说。目标是说清楚为什么这么想，不是证明自己是对的。

**写作规则（按优先级）：**

1. **前一段的指纹说了什么，照着写。** 这是最高优先级——比以下所有规则都高。指纹是从你真实的修改行为中提炼的，比任何规则都准。

2. **用具体的人/事/画面开头，不用抽象概括。** 不要写"随着……的发展"，写"李叔退休前是八级钳工"。

3. **一段只做一件事。** 不交代背景+描写人物+渲染情绪+暗示评论一起上。只做一件，放弃其他。

4. **句子控制在15字以内。** 用句号制造呼吸。连续三个长句后必须用一个短句断一下。

5. **事实放那里就行，不用补情绪。** 不要写"让我心里一沉""不禁深思"——直接把让你心里一沉的那个事实摆出来。

6. **不说自己不知道的事。** 不假装了解写触不到的领域。承认局限比假装全知有力。

7. **结尾留一个"悬空"。** 不用总结、不用升华、不用"这件事告诉我们"——用一句没法接话的短句收。

**段落产出标记：** 每段末尾标注字数（如 [298字]），便于判断段落节奏。

### 模块C：编辑Agent

**加载时机：** Phase 3

**任务：** 通读全文两遍。第一遍只感受不出手。第二遍逐段做以下三个动作：

**动作一：砍三分之一。** 删掉所有铺路搭桥的内容——概念解释、段落过渡、观点铺垫、读起来像在为你下结论做准备的内容。删完之后如果感觉断了——让断的地方断着，读者会自己跟上来。

**动作二：清除情感管理。** 逐句检查以下类别，找到就删（不要犹豫、不要评估）：
- "让我心里一沉""让人不禁深思""这背后折射出"——情感预告，删
- "那是一个无法言说的……"——情感命名，删
- "我不是说……""我不否认……"——防御声明，删
- "生活就是这样""人的感觉就是这样"——哲理升华，删
- "作为一个普通人……"——写作资格解释，删

**动作三：搜索以下表层隐患。** 找到就改：
- 破折号（—）超过1处 → 改成逗号或句号
- "该/其/之"出现（非指代用法）→ 换成"这个/他的/的"
- 冒号列表 → 改成口语叙述
- "首先/其次/最后"或"然后"过度 → 删
- "值得注意的是/不可忽视的是" → 删
- 感叹号超过1个 → 只保留最该保留的那个

**输出：** 修改后的全文 + 改了什么（一句话总结改动方向，不是逐条清单）。

### 模块D：统稿Agent

**加载时机：** Phase 4

**任务：** 初稿被编辑Agent修剪后，检查段与段之间的衔接。不补过渡词，不插"正如上文所述"。只做两件事：

1. **找语气断裂点：** 某一段突然换了说话节奏或视角——通知用户，不改，让用户决定。
2. **找重复信息：** 同一个故事/数据在两段里都提到了——删掉一次，保留信息量更多的那次。
3. **读开头三句话和结尾三句话——如果开头和结尾读起来像两个人写的，调整结尾的语气回开头的一致。**

**输出：** 统稿后的全文 + 改动记录（找出了几处断裂，删了几处重复）。

## 四、指纹系统

这是整个系统的核心。指纹不是规则，是从你真实的修改行为中生长出来的。

### 存放位置

```
.workbuddy/memory/writer-loop/fingerprint-{文章名}.md
```

一篇一个文件。新文章可以继承旧指纹（`cp` 命令复制），也可以从零开始。

### 指纹记录流程（自动）

每次你在 Phase 2 改完一段，我和指纹系统自动做三件事：

1. **对比：** AI原稿 vs 你的修改稿，逐句找出差异
2. **提炼：** 从差异中归纳风格偏移的方向（不是"改了哪些词"，而是"往哪个方向改的"）
3. **写入：** 追加到指纹文件

### 指纹文件格式

```markdown
# 指纹日志
## 文章：{文章名}
## 更新时间：{时间}

## 第X段修改记录

### AI原稿
[粘贴AI输出的原文]

### 你的修改稿
[粘贴你改完的版本]

### 提炼的风格偏移
- 开头方式：概括→具体/抽象→画面/观点→人物 ...
- 句子长度：由 xx→yy 字
- 信息偏好：宏观数据→微观个体
- 修辞方式：完整→断裂/说明→暗示

## 累计概况（基于X轮修改）
### 稳定偏好（出现3次以上）
✅ 偏好项1
✅ 偏好项2

### 有待观察（出现1-2次）
⚠️ 偏好项3

### 杂音（本次出现但与前几轮矛盾）
- 与之前模式冲突的修改（暂不加入稳定偏好）
```

### 指纹在写作中的使用

每次 Phase 2 启动写作Agent前，读取当前指纹文件，提取"稳定偏好"列表，**直接注入写作Agent的指令头部：**

```
【请按以下风格写作】
根据用户最近的修改习惯：
- 偏好用具体人物开头，替代宏观概括
- 句子平均12字，多用句号断句
- 段落以画面收尾，不加评论

请按此风格续写下一段。
```

指纹越厚，指令越准。写到最后几段时，Agent 的产出已经明显贴合你的手。

### 指纹继承（跨文章）

新文章开始时，可以选择继承上一篇的指纹文件：

```bash
# 继承写法
cp fingerprint-养老金第三篇.md fingerprint-婚姻数据新文章.md
```

然后继续在 Phase 2 中增厚。**指纹可以跨文章传承，系统越用越了解你。**

### 指纹蒸馏（可选）

当稳定偏好达到10条以上时，做一次蒸馏：将10条浓缩为3-5条核心规则，旧记录归档，指纹文件长度回落。蒸馏后的精简版作为新的默认指纹，旧版保存在 `fingerprint-{文章名}-archive.md` 中。

## 五、Token 预算说明

| 阶段 | 加载内容 | 预估Token |
|------|---------|-----------|
| Phase 0 | 模块A（~250字）| ~350 |
| Phase 1 | 热身四问（~300字）| ~400 |
| Phase 2 每段 | 模块B + 指纹文件（每轮增长100-200字）| ~800-1500 |
| Phase 3 | 模块C（~500字）| ~700 |
| Phase 4 | 模块D（~200字）| ~300 |
| Phase 5 | 无指令，仅调用产出 | ~50 |

一篇8段的文章，总Token消耗约 **8000-15000**。远低于一次性加载全部约束+技能文件（通常30000+）。原因：每轮只加载当前所需的最小模块。
